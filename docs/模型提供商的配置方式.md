# 模型提供商的配置方式

## Claude Code

**配置文件**：`~/.claude/settings.json`（优先），回退到 `~/.claude/claude.json`

**配置格式**：JSON，通过 `env` 字段设置环境变量

**核心字段**：

| 字段 | 作用 |
|------|------|
| `ANTHROPIC_BASE_URL` | 提供商 API 地址 |
| `ANTHROPIC_AUTH_TOKEN` / `ANTHROPIC_API_KEY` | 认证凭证 |
| `ANTHROPIC_MODEL` | 主模型 |
| `ANTHROPIC_DEFAULT_SONNET_MODEL` | Sonnet 角色模型 |
| `ANTHROPIC_DEFAULT_OPUS_MODEL` | Opus 角色模型 |
| `ANTHROPIC_DEFAULT_HAIKU_MODEL` | Haiku 角色模型 |
| `ANTHROPIC_DEFAULT_FABLE_MODEL` | Fable 角色模型 |

**写入方式**：直接写入 `~/.claude/settings.json`，使用原子写入保证一致性。切换提供商时，替换整个 `env` 字段。启用代理模式时，`ANTHROPIC_BASE_URL` 指向本地代理地址，API Token 替换为占位符。

**插件联动**：写入 `~/.claude/config.json`，设置 `primaryApiKey = "any"` 以允许第三方认证。

### 配置示例

**~/.claude/settings.json**

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "https://api.example.com",
    "ANTHROPIC_AUTH_TOKEN": "sk-ant-your-api-key",
    "ANTHROPIC_MODEL": "anthropic/claude-sonnet-5",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "anthropic/claude-sonnet-5",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "anthropic/claude-opus-4.8",
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "anthropic/claude-haiku-4.5"
  }
}
```

**~/.claude/config.json**（插件联动）

```json
{
  "primaryApiKey": "any"
}
```

## Codex

**配置文件**：两个文件协同工作

| 文件 | 格式 | 作用 |
|------|------|------|
| `~/.codex/auth.json` | JSON | 存储认证凭据（API Key 或 OAuth 令牌） |
| `~/.codex/config.toml` | TOML | 定义模型提供商、模型、API 格式 |

**config.toml 核心字段**：

| 字段 | 作用 |
|------|------|
| `model_provider` | 当前使用的提供商 ID（第三方统一为 `"custom"`） |
| `model` | 主模型名称 |
| `model_reasoning_effort` | 推理强度（low / medium / high） |
| `[model_providers.custom]` | 自定义提供商配置块 |
| `base_url` | 提供商 API 地址 |
| `wire_api` | API 格式（`"responses"` 或 `"chat_completions"`） |
| `requires_openai_auth` | 是否需要 OpenAI 认证 |
| `experimental_bearer_token` | 提供商级别的 bearer token |

**写入方式**：使用原子写入同时更新两个文件，先写 `auth.json` 再写 `config.toml`，失败时自动回滚。切换提供商时，若启用保留官方认证设置，则只更新 `config.toml`，不覆盖 `auth.json` 中的登录凭据。

### 配置示例

**自定义提供商 - ~/.codex/auth.json**

```json
{
  "OPENAI_API_KEY": "sk-your-api-key"
}
```

**自定义提供商 - ~/.codex/config.toml**

```toml
model_provider = "custom"
model = "gpt-5.5"
model_reasoning_effort = "high"
disable_response_storage = true

[model_providers.custom]
name = "example-provider"
base_url = "https://api.example.com/v1"
wire_api = "responses"
requires_openai_auth = true
```

## Codex 官方与自定义提供商切换

Codex 支持两种认证模式：**官方**（ChatGPT OAuth 登录）和**自定义**（第三方 API Key）。

### 官方提供商配置

使用 ChatGPT OAuth 认证，`config.toml` 为空，`auth.json` 存储 OAuth 令牌。

**~/.codex/auth.json**（官方）

```json
{
  "auth_mode": "chatgpt",
  "tokens": {
    "access_token": "oauth-access-token-id",
    "id_token": "jwt-identity-token"
  }
}
```

**~/.codex/config.toml**（官方）

空文件，使用 Codex 默认配置走 ChatGPT 官方端点。

### 切换行为

通过 "保留官方认证" 设置控制切换行为：

**官方 → 自定义**
- 启用保留：仅更新 `config.toml`，`auth.json` 中的 OAuth 凭据保持不变，随时可切换回官方
- 未启用保留：同时覆盖 `auth.json` 和 `config.toml`，OAuth 凭据被清除

**自定义 → 官方**
- 清空 `config.toml`
- 恢复 `auth.json` 中的 OAuth 凭据（来自系统 Keychain 或缓存）
