项目模块 UI 线框图 · 功能规划工作台

来源：项目功能开发文档.md；制作规则：UI线框图制作规范.md。本页把项目模块的常规界面、状态反馈、数据流和验收点集中到同一个 HTML 中，后续 PNG 应从每个 .shot-card 截图导出。

公共前提

新增“项目”主入口，配置、终端、编排三类现有 Tab 保持逻辑不变。项目 Tab 内采用左侧项目树、中部终端、右侧辅助抽屉的桌面工具布局。

Tauri 桌面端 Windows 优先 终端复用

线框风格

保持灰白工作台、深色顶栏、清晰边框和状态色。所有按钮、菜单、抽屉、Toast、编辑器和数据流在真实界面语境中表达，不做脱离界面的状态大集合。

结构优先 可截图卡片 左图右说明

负向要求

不新增文档外入口，不把右侧抽屉做成常驻面板，不让关闭终端等同删除会话，不混淆项目最近记录，不把导出 PNG 回填成线框源图。

无营销页 无重复 prompt 堆叠 无乱码

关闭语义
会话行负责展示状态圆点和会话名；关闭终端作为独立操作处理，关闭 PTY 时仍保留会话记录。

  • 删除会话是独立操作：需要先关闭 PTY，再删除元数据。
  • 状态判断复用终端运行态：无 PTY 为 off，2 秒内有输出为 running，否则 idle。

纯文本与不支持文件
txt/log/env 等纯文本按普通编辑器打开，仍显示行号和保存状态。

pdf/exe/图片等不在内置编辑器中打开，使用飘字提示。

说明
  • 代码文件支持常见语法高亮、行号和基础主题；纯文本文件按文本编辑器展示。
  • 无法打开的文件类型只给出清晰提示，不创建空编辑器。
  • 拖拽 md/txt/代码文件到右侧边栏或会话区域时，也会打开右侧文件编辑器并记录最近项目。

UI-WF-01 项目模块主界面默认状态 
中文：基于上传线框图生成桌面工具型项目模块 UI，严格遵守单层顶部应用栏、左侧 210px 项目树、中部终端、右侧默认空的布局；不要新增文档外按钮或营销装饰；保持灰白工作台、深色标题栏和清晰状态圆点。
English: Generate a desktop productivity UI from the uploaded wireframe. Strictly preserve the single top title bar, 210px project tree, central terminal, and no visible right drawer by default. Do not add extra buttons, banners, or decorative marketing elements.
UI-WF-02 左侧项目边栏操作 
中文：突出项目选择、折叠、项目行更多菜单、项目行最右侧新会话图标、文件夹新增和拖拽新增项目的交互状态；菜单必须贴近触发按钮；保持项目树可扫读。
English: Emphasize project selection, collapse, the project-row more menu, the far-right new-session icon on the expanded project row, add-folder, and drag-to-add states. Menus should appear near their trigger and the project tree should stay scannable.
UI-WF-03 项目会话生命周期 
中文：展示项目下的会话列表、当前会话切换和重命名状态；每个会话行左侧必须有灰/绿/橙状态圆点，右侧显示会话名称，不出现关闭按钮或终端标签页；展开项目行最右侧有新会话图标。
English: Show the project session list, active session switching, and rename state. Each session row must have a gray/green/orange status dot on the left and the session name on the right, without close buttons or terminal-tab styling. The expanded project row has a far-right new-session icon.
UI-WF-04 中部模块顶栏 
中文：中部模块顶栏只显示当前会话标题和一个右侧边栏展开按钮；不要显示项目名，不要做标签页，不要加入更多或新会话按钮。
English: The central module toolbar should only show the active session title and one right-sidebar toggle. Do not show the project name, tabs, more-actions buttons, or creation buttons.
UI-WF-05 右侧工具菜单与最近记录 
中文：右侧抽屉从中部内容区右边出现，工具菜单包含文件、终端、浏览器；最近记录必须标明当前项目，体现按项目隔离。
English: The right drawer should slide from the right side of the central content. Include File, Terminal, and Browser tools. Recent items must show the active project and imply project isolation.
UI-WF-06 Markdown 文件编辑器 
中文：展示右侧抽屉内 Markdown 编辑器，顶部是短标签页条，标签内关闭按钮紧跟标题，最右侧加号弹出打开功能列表；源码和预览通过按钮切换，不要并列展示；右侧边栏可接收 md/txt/代码文件拖拽并作为新标签打开。
English: Show a Markdown editor inside the right drawer. The top uses short tabs with each close button next to its title, and a plus button on the far right opens a small function menu. Source and preview switch through a button instead of being shown side by side. The right drawer accepts dropped md/txt/code files and opens them as new tabs.
UI-WF-07 代码文本与不支持类型 
中文：代码文件显示行号与基础语法高亮；纯文本可编辑；不支持文件类型用 toast 提示，不新增空白编辑面板。
English: Code files should show line numbers and basic syntax highlighting; plain text remains editable; unsupported file types should show a toast instead of creating an empty editor panel.
UI-WF-08 辅助终端 
中文：右侧终端是独立辅助实例，默认 cwd 为当前项目，不出现在左侧会话树；终端视觉与中部终端保持同源。
English: The right-side terminal is an independent helper instance with cwd set to the active project. It must not appear in the left session tree, and should visually match the central terminal.
UI-WF-09 内置浏览器 
中文：浏览器面板保留后退、前进、刷新和地址栏，展示空状态与 URL 加载后的页面骨架；不要新增复杂浏览器功能。
