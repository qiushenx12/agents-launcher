# 项目功能开发文档

> 对应线框图：`docs/UI线框图_项目功能规划.html`
> 适用范围：以“项目”模块为主；配置页的“使用内置终端”启动入口会接入项目会话，其余配置、终端、编排逻辑保持原有边界。

---

## 1. 总体目标

在 ClaudeEnvManager 中新增“项目”主入口。进入后呈现左中右三栏布局：

- **左侧**：项目列表 + 项目下会话树；设置入口移至项目列表最下方。
- **中部**：复用现有终端组件，每个会话对应一个终端实例。
- **右侧**：默认空；通过中部顶栏右侧的展开按钮唤出辅助边栏，支持打开文件、终端、浏览器。

顶部应用栏合并为单栏：左侧边栏折叠按钮 → 配置/项目 Tab → 窗口控制按钮。终端、编排相关代码保留，但不再展示为顶栏主入口。

## 2. 界面结构

### 2.1 顶部应用栏（App Title Bar）

只保留一层顶栏，高度约 36px，深色背景：

```
[☰] [配置] [项目]          [—] [□] [✕]
```

- **☰**：常驻显示；在配置功能内折叠/展开配置左侧栏，在项目功能内折叠/展开项目左侧栏。
- **配置 / 项目**：主功能入口；当前激活项高亮。
- **窗口控制**：最小化、最大化、关闭。

> 实现提示：Tauri 的 `data-tauri-drag-region` 需覆盖顶栏空白区域，但按钮区域应排除拖拽。

### 2.2 左侧项目边栏

宽度固定 210px，顶部直接贴顶栏，底部到窗口底部。

#### 2.2.1 头部

```
项目 ▼              [⋯] [📁+]
```

- **项目 ▼**：点击可切换当前激活项目（弹项目选择菜单）。
- **⋯**：打开项目功能菜单，包含项目排序规则：
  - **手动排序**：项目按手动顺序显示；长按项目行可抓取拖拽调整顺序，若长按时项目处于展开状态则先收起。
  - **时间排序**：项目按该项目下最新更新会话的时间倒序显示。
- **📁+（新增项目目录）**：点击调用 Windows 文件夹选择对话框，选择目录后新增项目。复用“配置”功能中的目录选择逻辑。

#### 2.2.2 项目树

```
▼ 📁 医院仓库系统                              [⋯] [新会话]
   ● 主终端
   ● 测试分支
   ● 旧会话备份
▶ 📁 泡泡台球
▶ 📁 Claude 启动器
```

**项目节点行为：**

- 点击项目行：展开/收起该项目下的会话列表。
- 项目图标：文件夹图标。
- 右键项目行：弹出上下文菜单（重命名、打开目录、删除项目）。
- 项目行右侧 `⋯`：显示当前项目更多菜单（重命名、打开目录、删除项目等）。
- 项目行最右侧“新会话”图标：在该项目下创建并打开新会话，逻辑等同项目 Tab 内 `Ctrl+T`。

**会话节点行为：**

- 点击会话行：在中部打开/切换到对应终端。
- 状态圆点：
  - 灰色 `●`：未开启（无活跃 PTY 进程）。
  - 绿色 `●`：已开启、待机中（PTY 存活但无输出）。
  - 橙色 `●`：运行中（近期有输出或命令执行中）。
- 展开项目下的会话列表初始展示 5 条，按最近编辑时间倒序排列；超过 5 条时在列表底部显示“展开显示”按钮，每次点击多展示 5 条。
- 当该项目会话已全部展示时，按钮文案切换为“折叠显示”；点击后恢复只展示 5 条。收起项目时也重置该项目会话列表，下次展开恢复初始 5 条。
- 会话行展示状态圆点、会话名称和右侧相对时间（距最近编辑时间，如 `4 小时`、`1 天`）；鼠标悬浮会话行时隐藏时间并显示关闭按钮。关闭终端是独立操作，关闭 PTY 后仍保留会话记录。

> 状态逻辑复用终端运行态：无 PTY 为未开启，PTY 存活但无近期输出为待机，近期有输出为运行中。

#### 2.2.3 底部设置

```
⚙ 设置
```

- 点击后打开设置弹窗/面板，内容与右上角原设置一致。
- 原 App.vue 右上角设置按钮移除。

### 2.3 中部终端区

复用现有 `TerminalManager` / 终端组件，但有以下调整：

- 中部顶栏只保留当前会话标题和右侧边栏展开按钮：

```
主终端                                                  [▯]
```

- **▯**：展开/收起右侧辅助边栏。
- 当前项目切换、更多操作、新建会话、新增项目目录等入口只出现在左侧项目边栏。
- 不在中部顶栏做终端标签页、项目切换、更多操作或新会话按钮。
- 项目行最右侧“新会话”图标或 `Ctrl+T` 均可新建会话；两者复用“配置”功能里的“启动 Claude Code”逻辑，在当前项目目录下创建并打开新的会话终端。
- 配置页勾选“使用内置终端”后点击“启动 Claude Code”或点击“最近会话”，不再跳转到普通“终端”功能；应按启动目录找到/新增对应项目，在左侧项目会话列表创建或激活入口，并跳转到“项目”功能启动中部项目终端。
- `Ctrl+W` 关闭当前活动会话的 PTY，左侧会话记录仍存在。
- 中部终端实例按 `sessionId` 索引，切换会话时保留状态。

### 2.4 右侧辅助边栏

**默认状态：** 右侧没有任何 UI，中部内容占满剩余宽度。

**展开触发：** 点击中部顶栏最右侧 `▯` 按钮。

**展开行为：** 宽度约 260px 的抽屉从**中部内容区右侧**滑入，覆盖或挤压中部内容（推荐覆盖，不压缩终端宽度）。

#### 2.4.1 顶部标签与工具菜单

展开后顶部为短标签页条，每个打开项都是一个标签，关闭按钮在标签内紧跟标题；最右侧固定为 `+` 按钮，点击后弹出打开功能列表：

```
备忘录.md ×   终端 ×   claude.ai ×                 +
─────────────────────────
打开
  📄 文件                    Ctrl+P
  🖥 终端                    Ctrl+T
  🌐 浏览器                  Ctrl+B
```

- 点击标签切换右侧边栏当前面板。
- 点击标签内 `×` 只关闭该右侧边栏标签；文件 dirty 时先提示保存。
- 点击 `+` 展示小列表，可打开文件、辅助终端、浏览器。
- 默认展开边栏时显示工具菜单；**“最近”列表按当前激活项目隔离**：

```
─────────────────────────
最近（医院仓库系统）
  📝 备忘录.md
  🌐 claude.ai
```

- **文件**：打开系统文件对话框，选择文件后在边栏内打开编辑器。
- **终端**：在当前项目目录下新建一个辅助终端会话。
- **浏览器**：在边栏内打开浏览器面板。
- **最近**：仅显示当前项目最近打开过的文件、浏览器 URL、辅助终端；切换项目后列表随之切换。

#### 2.4.2 文件编辑器面板

```
备忘录.md ×
─────────────────────────
源码 | 预览
─────────────────────────
1  # 备忘录
2
3  - 待办事项
```

- **Markdown 文件**：使用 Markdown 语法高亮，源码/预览通过顶部按钮切换，同一时间只展示一个视图。
- **代码文件**：支持常见代码文件语法高亮（js/ts/vue/html/css/rust/python/json/yaml 等），提供行号、基础主题。
- **纯文本文件**：按文本编辑器展示。
- 标题显示修改状态（未保存加点或“已修改”）。
- 快捷键 `Ctrl+S` 保存。
- 无法打开的文件类型（如 pdf、exe）飘字提示：`无法打开该文件类型：xxx`。
- 打开文件时自动加入当前项目的 `recentItems` 并置顶。

#### 2.4.3 辅助终端面板

与中部终端共用同一套 PTY 逻辑，但独立实例：

```
终端 ×
─────────────────────────
$ cd D:\Work\hospital-warehouse
$ _
```

- 默认工作目录为当前选中项目目录。
- 不显示在项目左侧会话树中。

#### 2.4.4 浏览器面板

```
浏览器 ×
[◀] [▶] [↻] [    输入 URL...    ]
─────────────────────────
        🌐
      开始浏览
   输入 URL 以打开页面
```

- 使用 Tauri Webview 或 `<iframe>` 加载网页。
- 地址栏输入 URL 回车后加载。
- 前进/后退/刷新基本导航。

## 3. 数据模型

### 3.1 Project（项目）

```typescript
interface Project {
  id: string              // uuid
  name: string            // 显示名称
  path: string            // 项目绝对路径
  createdAt: number
  updatedAt: number
  order: number           // 排序权重
  recentItems: RecentItem[] // 该项目在右侧边栏最近打开的文件/URL/终端记录
}

interface RecentItem {
  type: 'file' | 'browser' | 'terminal'
  name: string            // 显示名称
  path?: string           // 文件绝对路径（type=file 时必填）
  url?: string            // 浏览器地址（type=browser 时必填）
  openedAt: number        // 最近打开时间，用于排序
}

interface SidebarTab {
  id: string
  type: 'tools' | 'file' | 'terminal' | 'browser'
  title: string
  path?: string           // type=file 时使用
  url?: string            // type=browser 时使用
  terminalId?: string     // type=terminal 时使用
  content?: string        // 文本文件内容
  dirty?: boolean         // 文件是否未保存
  viewMode?: 'source' | 'preview' // Markdown 文件视图
}
```

### 3.2 ProjectSession（会话）

```typescript
interface ProjectSession {
  id: string
  projectId: string
  name: string            // 会话名称
  claudeSessionId?: string // Claude Code sessionId，用于 -r 恢复最近会话
  shell?: string[]        // 启动命令，默认 ['cmd.exe']
  cwd?: string            // 启动目录，默认项目 path
  env?: Record<string, string>
  createdAt: number
  order: number
}
```

### 3.3 终端运行状态（复用现有）

```typescript
type TerminalStatus = 'off' | 'idle' | 'running'

interface TerminalRuntimeState {
  sessionId: string
  status: TerminalStatus
  ptyId?: string          // 后端 PTY 会话标识
  lastOutputAt?: number   // 用于判断 idle / running
}
```

### 3.4 应用启动 Tab 状态

```typescript
interface AppLaunchState {
  lastActiveMainTab: 'config' | 'terminal' | 'project' | 'orchestration'
  // 记录关闭时是否处于项目功能；非项目时默认进入配置
}
```

关闭 App 时持久化当前激活的主 Tab；下次启动时：
- 若 `lastActiveMainTab === 'project'`，直接进入项目 Tab。
- 否则进入**配置** Tab。

## 4. 状态管理（Pinia）

新增 `src/stores/project.ts`：

```typescript
export const useProjectStore = defineStore('project', () => {
  // State
  const projects = ref<Project[]>([])
  const sessions = ref<ProjectSession[]>([])
  const expandedProjectIds = ref<Set<string>>(new Set())
  const activeProjectId = ref<string | null>(null)
  const activeSessionId = ref<string | null>(null)
  const sidebarOpen = ref(false)        // 右侧边栏是否展开
  const sidebarTabs = ref<SidebarTab[]>([])
  const activeSidebarTabId = ref<string | null>(null)

  // Getters
  const activeProject = computed(() => projects.value.find(p => p.id === activeProjectId.value))
  const sessionsOfActiveProject = computed(() =>
    sessions.value.filter(s => s.projectId === activeProjectId.value).sort((a, b) => a.order - b.order)
  )
  const recentItemsOfActiveProject = computed(() =>
    (activeProject.value?.recentItems || []).sort((a, b) => b.openedAt - a.openedAt)
  )

  // Actions
  function loadProjects()
  function addProject(path: string, name?: string)
  function removeProject(id: string)
  function renameProject(id: string, name: string)
  function toggleProjectExpanded(id: string)
  function createSession(projectId: string, name?: string) // 项目行新会话按钮或 Ctrl+T 调用
  function launchClaudeFromConfig(sessionId?: string)      // 配置页内置终端入口：按启动目录进入项目并启动项目会话
  function removeSession(id: string)           // 仅删除记录，若运行中先关闭 PTY
  function activateSession(id: string)
  function closeSessionTerminal(id: string)    // 只关终端，保留记录
  function openSidebar(defaultTab?: SidebarTab['type'])
  function closeSidebar()
  function openSidebarTab(type: SidebarTab['type'], payload?: string)
  function closeSidebarTab(id: string)
  function openFile(path: string)              // 作为右侧边栏标签打开，并更新当前项目的 recentItems
  function saveFile(tabId: string)
  function setFileViewMode(tabId: string, mode: 'source' | 'preview')
  function recordRecentItem(type: RecentItem['type'], name: string, payload?: string) // 更新最近记录
})
```

### 5.4 终端复用策略

中部终端复用现有 `TerminalManager`，但需要支持按 `sessionId` 管理多个终端实例：

- `TerminalManagerForProject` 接收 `activeSessionId`。
- 内部维护 `Map<sessionId, TerminalInstance>`，切换 session 时显示对应实例。
- 终端实例销毁时机：
  - 用户按 `Ctrl+W` 或触发“关闭当前终端”操作：关闭 PTY，实例释放，但 session 记录保留。
  - 用户删除会话：先关闭 PTY 再删除记录。

> 若现有 `TerminalManager` 强绑定于 `tabId`，可封装一层适配器，将 `sessionId` 映射为内部 `tabId`。

## 7. 交互细节

### 7.1 拖拽添加项目/文件

- 拖拽目录到左侧项目列表空白处：新增项目。
- 拖拽目录到已有项目上：询问是否替换项目路径或作为子目录打开。
- 拖拽文件到已展开的右侧边栏，或拖到左侧会话区域：
  - md/txt/受支持代码文件：在右侧边栏打开文件编辑器；若边栏未展开，先展开边栏再打开。
  - 其他类型：飘字 `无法打开该文件类型：xxx.ext`。

### 7.2 会话名称

- 新建会话默认名：`终端 1`、`终端 2`… 或 `新会话 1`。
- 右键会话可重命名。
- 会话名回车保存，失焦保存，Esc 取消。
- 会话行右侧时间来自 `updatedAt`，展示距最近编辑时间的相对值。

### 7.3 状态判断

```typescript
function getSessionStatus(sessionId: string): TerminalStatus {
  const pty = ptyStore.getPty(sessionId)
  if (!pty || !pty.alive) return 'off'
  if (Date.now() - pty.lastOutputAt < 2000) return 'running'
  return 'idle'
}
```

- `running` 可通过监听终端输出事件刷新 `lastOutputAt` 实现。
- 可定时轮询，每 2 秒根据 `lastOutputAt` 重新计算。

### 7.4 右侧边栏打开文件

- 通过 Tauri `open` 对话框选择文件。
- 也支持将 md/txt/受支持代码文件直接拖拽到右侧边栏空白处、标签栏或编辑器区域，作为新标签打开。
- Rust 读取文本内容返回前端。
- 编辑器内修改时标记 `dirty=true`。
- `Ctrl+S` 触发 `save_file` 命令写回磁盘。
- 关闭编辑器前若 `dirty=true` 提示保存。
- **打开文件后**：调用 `recordRecentItem('file', fileName, filePath)`，将记录追加到当前项目的 `recentItems` 并排序置顶。

### 7.5 右侧边栏浏览器

- 输入 URL 后通过 Tauri `WebviewWindow` 或 `<iframe>` 加载。
- **打开浏览器后**：调用 `recordRecentItem('browser', url, url)`，将 URL 记录到当前项目的 `recentItems`。
- 考虑到安全性与隔离，推荐用 `<iframe>` 嵌套在边栏内；如需系统级浏览器体验，可调用 `shell::open` 打开系统浏览器，但与本功能“内置”定位不符。

## 9. 持久化策略

1. **项目/会话元数据 + 最近记录**：每次增删改后保存到 `projects.json`；`recentItems` 按项目隔离。
2. **窗口状态**：继续由 `persistent_state` 负责。
3. **右侧边栏展开状态与打开标签**：不持久化，启动后默认收起；最近记录持久化。
4. **会话终端输出**：不持久化，关闭后清空。
5. **未保存文件**：关闭前弹窗提示，或自动保存草稿到 `%APPDATA%\ClaudeEnvManager\drafts\`。
6. **最后激活的主 Tab**：关闭 App 时持久化 `lastActiveMainTab`；下次启动时若为 `project` 则进入项目，否则进入配置。
7. **Claude history 变更同步**：监听 `~/.claude/history.jsonl` 变化，变更后立即刷新配置页最近会话、启动目录历史和项目会话列表；刷新保留“加载更多/展开显示”的当前展示数量。

## 10. 依赖与实现风险

### 10.1 依赖

- 前端：Markdown 源码/预览切换可考虑 `markdown-it` + `highlight.js`；代码语法高亮使用 `highlight.js` 或 `prismjs`。按需引入，避免打包过大。
- 后端：Tauri `dialog` plugin（已具备文件选择能力）；`notify` 用于监听 Claude `history.jsonl` 变更并通知前端刷新。
- 浏览器：如需内嵌，使用原生 Webview；如需更强控制，考虑 `@tauri-apps/plugin-shell`。

### 10.2 风险与注意点

1. **终端状态同步**：中部终端与左侧会话状态需双向同步。确保 `pty` 事件能正确更新对应 `sessionId` 的状态。
2. **多实例内存**：长期运行会积累大量未关闭终端实例。建议在会话关闭时彻底释放 PTY。
3. **路径跨平台**：当前仅 Windows，但路径拼接仍需使用 Rust `std::path`。
4. **拖拽文件类型**：Tauri 拖拽事件需启用 `fs:allow-read` 并校验路径。
5. **浏览器安全**：`<iframe>` 加载外部网页存在 XSS 风险，建议仅允许用户主动输入的 URL，不做自动加载。
